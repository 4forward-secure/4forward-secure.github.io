  • :: [1] Introduction

    Attention: It is worth remembering that it is extremely important to save the information obtained in the recognition phase in a notepad, as you can be sure that you will use it later on.


    Hypothetically, consider that we are targeting the website https://www.tango.com/ and we want to perform reconnaissance on this target, finding subdomains, important directories, files, social networks, and publicly accessible information. This phase is of utmost importance as it will define how and where our possible entry points will be.


    We have two paths to follow, passive or aggressive reconnaissance. We will always start by using the passive path, as we do not want to alert the target about our presence and intentions.


    In this endeavor, your best ally will be Google and/or other search engines, either to find other sites that perform target reconnaissance or to conduct advanced searches yourself.


    I will talk more about Google, as it is the most common and famous, but below you will find documentation for other search engines.

    1. DuckDuckGo Search Syntax

    2. Bing Advanced Search

  • :: [2] Google Hacking

    The most common search engine for advanced searches is Google, along with a famous technique called Google Hacking. Below are some examples of Google dorks that we can use to obtain some information. 


    +-----------------+-------------------+--------------------------------------------+
| Operator/Syntax | Example           | Function                                   |
+-----------------+-------------------+--------------------------------------------+
| Site            | site:tango.com    | Search for a specific site.                |
+-----------------+-------------------+--------------------------------------------+
| Related         | related:tango.com | Search for related sites.                  |
+-----------------+-------------------+--------------------------------------------+
| ""              | "phrase or word"  | Find results with exact search phrase.     |
+-----------------+-------------------+--------------------------------------------+
| filetype        | filetype:pdf      | Find PDF files related to a certain term.  |
+-----------------+-------------------+--------------------------------------------+
| inurl           | inurl:tango.com   | Search for URLs with such specific words.  |
+-----------------+-------------------+--------------------------------------------+
| intext          | intext:gmail.com  | Search for specific words in source code.  |
+-----------------+-------------------+--------------------------------------------+

    If you perform a search as illustrated below, you will see that with just a simple Google search, we can list some tango subdomains. 


    +--------------------------=BROWSER=-------------------------+

  • :: [3] crt.sh

    By directing our reconnaissance towards gathering valuable information about subdomains, we can take advantage of the resources of the https://crt.sh/ website. This website offers a unique service that allows us to delve into the information on our tango's SSL certificate. By doing so, we can discover any domains that are directly associated with their certificate.

  • :: [4] dnsrecon

    When we talk about dns, the first tool that should come to mind is dnsrecon; with this tool, we can do all kinds of domain listing, from searches on bing, yandex, crt.sh, to even brute force dictionaries.


    As mentioned above, you can also use the dnsrecon tool to query crt.sh. I prefer to use dnsrecon for its simplicity and practicality, rather than the website.

  • :: [5] Wappalyzer

    We can use Wappalyzer to find out which front/back-end frameworks the target is running on that site. This is simple information, but it will be useful in our future goals, as it will allow us to test for another range of vulnerabilities.


    Wappalyzer is a browser extension, available for download from the Google Chrome and Firefox stores.


    This extremely useful extension returns data such as the server's operating system, system version, page programming language and so on.

  • :: [6] Shodan

    Using the server IP or DNS obtained using the dnsrecon tool, we can search the internet for that particular IP. For this process we have a great ally, Shodan, also known as the hacker's Google.


    If you go to https://shodan.io, you'll see that there's a search field, just enter the IP address and you'll see a range of information appear. Information such as open ports, possible vulnerabilities and you don't even have to communicate directly with the server.
