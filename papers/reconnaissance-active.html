  • :: [1] Introduction

    For a reconnaissance to be considered active, it is necessary to establish some type of contact with the target, which can range from social engineering techniques to a direct connection with the target system.


    Active reconnaissance begins when there are direct connections to the target machine, these are actions that leave information in the system logs and can be identified as a possible intrusion.

  • :: [2] Ping

    The purpose of the ping command is to check whether a machine is online, generally passing IP (Protocol Address) or domain name as an argument.


    Ping sends an ICMP packet to a system and the system responds by sending back an ICMP Echo Reply if the packet is not blocked by any firewall (network security device that monitors incoming and outgoing network traffic). 


    +-----------------------------=TERMINAL=--------------------------------+
| hax1zi@4forward:~$ ping -c 3 google.com                               |

| 64 bytes from 2800:3f0:4001:838::200e: icmp_seq=2 ttl=120 time=51.6 ms|
| 64 bytes from 2800:3f0:4001:838::200e: icmp_seq=3 ttl=120 time=52.2 ms|

  • :: [4] Traceroute

    The function of traceroute is to trace the route taken by the packet from your system to another host, thus finding the IP addresses of the routers or hops that pcato traverses.


    The command also reveals the number of routers between the two systems. However, the route taken by packets may change because some routers use dynamic routing protocols that adapt to network changes.


    The parts in red are IP addresses or domains that I created

  • :: [5] Nmap

    Active reconnaissance with Nmap is a technique used to discover active devices on a network and information about them, such as operating systems, running services and open ports.


    -sS: This argument indicates a SYN scan (does not complete a connection) so it is a quick and silent scan

    -O: This argument serves to try to identify the target's operating system


    The parts in red are IP addresses or domains that I created


    +---------------------=TERMINAL=-------------------+

  • :: [6] DNSRecon

    DNSrecon is an open source tool generally used to discover information about the DNS record (Domain Name System - System that translates domain names to IP addresses).


    In active reconnaissance we use DNSrecon to explore the DNS of a domain or network and thus map subdomains, enumerate hosts and collect DNS records.


    The parts in red are IP addresses or domains that I created

  • :: [7] Conclusion

    General reconnaissance is a very important part of an attack and active reconnaissance is the last part of reconnaissance for the next stage.


    However, remember that active reconnaissance generates logs, and some techniques can be identified as attacks. Therefore, carry out this step with caution and responsibility.
