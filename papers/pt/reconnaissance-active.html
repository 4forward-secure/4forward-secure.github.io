  • :: [1] Introdução

    Para que um reconhecimento seja considerado ativo é necessário estabelecer algum tipo de contato com o alvo, que pode variar desde técnicas de engenharia social até uma conexão direta com o sistema alvo.


    O reconhecimento ativo começa quando há conexões diretas com a máquina alvo, são ações que deixam informações nos logs do sistema e podem ser identificadas como uma possível intrusão.

  • :: [2] Ping

    O objetivo do comando ping é verificar se uma máquina está online, geralmente passando um endereço IP (endereço de protocolo de Internet) ou nome de domínio como argumento.


    O Ping envia um pacote ICMP para um sistema e o sistema responde enviando de volta uma resposta de eco ICMP se o pacote não estiver bloqueado por nenhum firewall (dispositivo de segurança de rede que monitora o tráfego de entrada e saída da rede). 


    +-----------------------------=TERMINAL=--------------------------------+
| hax1zi@4forward:~$ ping -c 3 google.com                               |
| PING google.com (2800:3f0:4001:838::200e) 56 data bytes               |
| 64 bytes from 2800:3f0:4001:838::200e: icmp_seq=1 ttl=120 time=51.7 ms|
| 64 bytes from 2800:3f0:4001:838::200e: icmp_seq=2 ttl=120 time=51.6 ms|
| 64 bytes from 2800:3f0:4001:838::200e: icmp_seq=3 ttl=120 time=52.2 ms|
|                                                                       |
| --- google.com ping statistics ---                                    |
| 3 packets transmitted, 3 received, 0% packet loss, time 2002ms        |
| rtt min/avg/max/mdev = 51.576/51.844/52.219/0.273                   |

  • :: [4] Traceroute

    A função do traceroute é rastrear a rota percorrida pelo pacote do seu sistema para outro host, encontrando assim os endereços IP dos roteadores ou saltos que ele percorre.


    O comando também revela o número de roteadores entre os dois sistemas. No entanto, a rota seguida pelos pacotes pode mudar porque alguns roteadores usam protocolos de roteamento dinâmico que se adaptam às mudanças na rede.


    As partes em vermelho são endereços IP ou domínios que eu criei.

  • :: [5] Nmap

    O reconhecimento ativo com Nmap é uma técnica usada para descobrir dispositivos ativos em uma rede e informações sobre eles, como sistemas operacionais, serviços em execução e portas abertas.


    -sS: Este argumento indica uma varredura SYN (não completa a conexão), portanto é uma varredura rápida e silenciosa.

    -O: Este argumento serve para tentar identificar o sistema operacional do alvo.


    As partes em vermelho são endereços IP ou domínios que eu criei.


    +---------------------=TERMINAL=-------------------+
| hax1zi@4forward:~$sudo nmap -sS hax1zi.com       |
| ...                                              |
| Not shown: 996 filtered tcp ports (no-response)  |
| PORT     STATE SERVICE                           |
| 80/tcp   open  http                              |
| 443/tcp  open  https                             |
| 8080/tcp open  http-proxy                        |
| 8443/tcp open  https-alt                         |
| ...                                              |

  • :: [6] DNSRecon

    DNSrecon é uma ferramenta de código aberto geralmente usada para descobrir informações sobre o registro DNS (Domain Name System - Sistema que traduz nomes de domínio em endereços IP).


    No reconhecimento ativo utilizamos o DNSrecon para explorar o DNS de um domínio ou rede e assim mapear subdomínios, enumerar hosts e coletar registros DNS.


    As partes em vermelho são endereços IP ou domínios que eu criei. 


    +-------------------------------=TERMINAL=-------------------------------+
| hax1zi@4forward:~$ dnsrecon -d hax1zi.com                              |
| [*] std: Performing General Enumeration against: hax1zi.com...         |
| [-] All nameservers failed to answer the DNSSEC query for hax1zi.com   |
| [*] 	 SOA ns1.hax1zi.com xxx.xxx.xx.xx                                |
| [*] 	 NS ns4.hax1zi.com xxx.xxx.xx.xx                                 |
| [*] 	 NS ns3.hax1zi.com xxx.xxx.xx.xx                                 |
| [*] 	 NS ns1.hax1zi.com xxx.xxx.xx.xx                                 |
| ...                                                                    |

  • :: [7] Conclusão

    O reconhecimento em geral é uma parte muito importante de um ataque e o reconhecimento ativo é a última parte do reconhecimento para a próxima fase.


    Porém, lembre-se que o reconhecimento ativo gera logs e algumas técnicas podem ser identificadas como ataques. Portanto, execute esta etapa com cautela e responsabilidade.
